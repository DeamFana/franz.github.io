## 1. 系统架构

### 1.1 混合部署模式

系统支持两种运行模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **开发模式** | Next.js 服务器 + API | 开发调试、需要动态更新 |
| **静态导出** | 纯静态 HTML | 离线使用、分发给他人 |

## 3. 核心模块

### 3.1 useResumeData Hook

负责数据的统一获取：

```typescript
// 伪代码
function useResumeData() {
  // 1. 优先读取嵌入数据（静态HTML）
  if (window.__RESUME_DATA__) {
    return window.__RESUME_DATA__;
  }
  
  // 2. 尝试 API（开发模式）
  const response = await fetch('/api/resume');
  return response.json();
}
```

### 3.2 EditContext

管理编辑状态：

- `data` - 当前简历数据
- `isDirty` - 是否有未保存的更改
- `saveData()` - 保存到 API
- `generateAndDeploy()` - 构建静态页面
- `refreshData()` - 刷新最新数据

### 3.3 API 路由

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/resume` | GET | 读取 resume.json |
| `/api/resume` | POST | 写入 resume.json |
| `/api/generate` | POST | 执行静态构建 |

## 6. 优缺点

### 优点
- **灵活性高**：开发用 API，生产用静态
- **用户体验好**：编辑后即时预览
- **离线可用**：静态导出无需服务器

### 缺点
- 静态导出需要手动构建
- 两套配置需要维护
