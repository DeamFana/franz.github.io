## 1. 系统架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                   Frontend                       │
│                 (Next.js 14 SSG)                │
├─────────────────────────────────────────────────┤
│                   Data Layer                     │
│              (resume.json 静态数据)               │
├─────────────────────────────────────────────────┤
│                  Edit Mode                       │
│         (本地开发环境，仅开发模式可用)             │
├─────────────────────────────────────────────────┤
│                   Deploy                         │
│              (GitHub Pages)                      │
└─────────────────────────────────────────────────┘
```

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Next.js 14 (App Router) |
| 样式 | Tailwind CSS |
| 数据 | JSON 文件 (SSG) |
| 编辑 | React Hook Form + Zod |
| 部署 | GitHub Pages |

## 3. 部署架构

| 环境 | 用途 | 平台 |
|------|------|------|
| 开发 | 本地开发 | localhost |
| 生产 | 正式上线 | GitHub Pages |

### 3.1 GitHub Pages 部署流程

```
1. 本地修改 resume.json 数据
2. 运行 npm run build
3. git push origin main
4. GitHub Actions 自动部署
5. GitHub Pages 自动更新
```

---

## 4. 模式说明

### 4.1 展示模式（默认）

- 读取 `resume.json` 静态数据
- 构建时生成静态 HTML
- 部署到 GitHub Pages

### 4.2 编辑模式（仅开发环境）

- 本地开发时可通过密码进入
- 修改数据后保存到 `resume.json`
- 保存后自动重新构建

### 4.3 后端模式（预留）

- 未来有服务器后部署
- 提供在线编辑 API
- 数据存储到数据库
